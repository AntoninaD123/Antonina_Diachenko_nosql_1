### Частина 1
### 1. Чому аудіо-характеристики винесені в окремий об’єкт `audio_features`?

Аудіо-характеристики об’єднані в окремий об’єкт, тому що вони належать до однієї логічної групи параметрів треку. Це робить структуру документа більш зрозумілою та зручною для аналізу. Недолік — складніші індекси та запити по вкладених полях. Але працювати буде.

### 2. Чому виконавці зберігаються як масив, а не як рядок?

Це зручніше, бо масив дозволяє зберігати кількох виконавців для одного треку, також легше шукати. Наприклад, можна легко знайти всі треки конкретного артиста без обробки текстового рядка.

### 3. Що таке `$out` і чим він відрізняється від `$merge`?

`$out` записує результат агрегації в нову колекцію та повністю її перезаписує. `$merge` дозволяє оновлювати або додавати документи без повного перезапису. У цьому завданні використано `$out`, оскільки колекція `tracks` створювалась заново.
